[ Git 명령어 모음 ]


1. 설정 (Configuration)

git config : Git 환경 설정 조회 및 변경
* git config --global user.name "이름" : 전역으로 사용할 사용자 이름 설정
* git config --global user.email "이메일" : 전역으로 사용할 사용자 이메일 설정
* git config --list : 현재 적용된 모든 설정값(이름, 이메일, 에디터 등) 확인
* git config user.name : 현재 설정된 이름만 출력 (설정 확인용)
* git config --global core.editor "code --wait" : Git의 기본 에디터를 VS Code로 변경 (커밋 메시지 작성, Rebase 시 편리)
* git config --global alias.[단축어] [명령어] : 긴 명령어를 단축어로 등록 (예: git config --global alias.co checkout)


2. 시작하기 (Setup)
 
git init : 현재 디렉토리를 Git 저장소로 초기화 (.git 폴더 생성)
git clone [URL] : 원격 저장소의 내용을 내 컴퓨터로 복제
* git clone [URL] . : (꿀팁) 현재 폴더(빈 폴더일 때)에 하위 폴더를 만들지 않고 파일들을 바로 쫙 깔아줌
* git clone -b [브랜치명] [URL] : 특정 브랜치만 골라서 클론


3. 상태 확인 및 변경사항 감지 (Inspect)

git status : 작업 디렉토리의 파일 상태 확인 (수정됨, 스테이징됨, 추적 안 됨 등)
* git status -s : 파일 상태를 짧게 요약해서 출력 (Short 모드)

git diff : 파일의 구체적인 변경 내용(코드) 비교
* git diff : add 하기 전, 작업 디렉토리와 최근 커밋 간의 차이 확인
* git diff --staged : add 한 후, 커밋할 예정인 파일들의 변경 내용 확인
* git diff [브랜치A] [브랜치B] : 두 브랜치 간의 차이점 비교
* git diff --name-only : 코드는 제외하고, 변경된 파일의 이름 목록만 확인

git blame [파일명] : (범인 찾기) 파일의 각 줄을 누가, 언제, 어떤 커밋으로 수정했는지 표시


4. 스테이징 및 커밋 (Snapshot)

git add : 파일을 스테이징 영역(장바구니)에 추가
* git add . : 변경된 모든 파일(삭제 포함)을 스테이징
* git add [파일명] : 특정 파일만 콕 집어서 스테이징
* git add -p : (고수용) 파일 하나를 통째로 올리지 않고, 변경된 코드 덩어리(Hunk)별로 보고 선택해서 스테이징

git commit : 스테이징된 변경 사항을 확정하여 기록
* git commit -m "메시지" : 에디터를 열지 않고 커밋 메시지를 바로 작성하여 커밋
* git commit --amend : 방금 한 최신 커밋을 수정 (빠뜨린 파일 추가하거나 메시지 수정)
* git commit --amend -m "새 메시지" : 방금 한 커밋의 메시지를 바로 덮어쓰기
* git commit --amend --no-edit : 메시지는 그대로 두고, 내용만 수정(파일 추가 등)


5. 히스토리 조회 (Log)

git log : 전체 커밋 히스토리를 시간 순으로 조회
* git log -n : 최근 n개의 커밋만 보기 (예: git log -3)
* git log -p : 각 커밋의 변경된 코드 내용(diff)까지 상세 표시
* git log --oneline : 커밋 해시와 제목만 한 줄로 요약해서 표시
* git log --graph : 브랜치가 갈라지고 합쳐지는 흐름을 그래프로 표시
* git log --oneline --graph --all : (추천) 모든 브랜치의 흐름을 한눈에 시각화
* git log -- [파일명] : 특정 파일의 변경 이력만 필터링해서 확인
* git log --follow [파일명] : 파일 이름이 중간에 바뀌었어도 이전 기록까지 추적
* git log --author="이름" : 특정 작업자가 남긴 커밋만 검색
* git log --grep="키워드" : 커밋 메시지에 특정 단어가 포함된 커밋 검색

git reflog : HEAD가 이동한 모든 기록 조회 (실수로 지운 커밋이나 브랜치 복구용)


6. 브랜치 관리 (Branching)

git branch : 로컬 브랜치 목록 조회
* git branch -r : 원격(Remote) 브랜치 목록 조회
* git branch -a : 로컬과 원격의 모든 브랜치 조회
* git branch -vv : 로컬 브랜치가 어떤 원격 브랜치와 연결되어 있는지 확인
* git branch -m [새이름] : 현재 브랜치의 이름을 변경
* git branch -d [브랜치명] : 병합이 완료된 브랜치 삭제
* git branch -D [브랜치명] : 병합 여부와 상관없이 강제 삭제

git switch : 브랜치 이동 (Checkout의 이동 기능 분리)
* git switch [브랜치명] : 해당 브랜치로 이동
* git switch -c [새브랜치] : 새 브랜치를 생성하고 즉시 이동
* git switch - : 바로 직전에 있었던 브랜치로 이동


7. 병합 및 동기화 (Merge & Rebase)

git merge : 다른 브랜치의 내용을 현재 브랜치로 합침
* git merge [대상브랜치] : 대상 브랜치를 현재 브랜치에 병합
* git merge --no-ff [대상브랜치] : Fast-forward가 가능해도 강제로 병합 커밋을 남김 (히스토리 보존)
* git merge --abort : 병합 중 충돌(Conflict) 발생 시 병합을 취소하고 되돌림

git rebase : 브랜치의 베이스(시작점)를 재설정하여 히스토리를 일직선으로 만듦
* git rebase [대상브랜치] : 내 브랜치의 변경 사항을 대상 브랜치의 최신 커밋 뒤로 이어 붙임
* git rebase -i HEAD~[숫자] : (성형수술) 최근 커밋 n개를 하나로 합치거나(Squash), 메시지를 수정하는 등 대화형 모드 실행
* git rebase --abort : 리베이스 진행 중 취소

git cherry-pick [커밋해시] : 다른 브랜치의 특정 커밋 하나만 복사해서 현재 브랜치로 가져옴


8. 원격 저장소 제어 (Remote)

git remote : 원격 저장소 연결 관리
* git remote -v : 연결된 원격 저장소의 주소(URL) 확인
* git remote add origin [URL] : 원격 저장소 주소 추가
* git remote remove origin : 기존 연결 삭제
* git remote set-url origin [새URL] : 원격 저장소 주소 변경

git push : 로컬의 변경 사항을 원격으로 업로드
* git push -u origin [브랜치명] : 업스트림(추적)을 설정하며 푸시 (최초 1회 필수)
* git push origin [브랜치명] : 지정한 브랜치 푸시
* git push --all : 로컬의 모든 브랜치를 원격으로 푸시
* git push --force (또는 -f) : 원격 내용을 무시하고 강제로 덮어쓰기 (위험)
* git push --force-with-lease : (안전) 다른 사람이 쓴 코드가 없을 때만 강제 푸시 허용
* git push origin --delete [브랜치명] : 원격 저장소의 브랜치 삭제

git fetch : 원격 저장소의 최신 이력을 가져오기만 함 (병합 X)
* git fetch --prune : 원격에서 삭제된 브랜치를 내 로컬 목록에서도 정리

git pull : 원격의 내용을 가져와서 합침 (Fetch + Merge)
* git pull origin [브랜치명] : 원격 브랜치를 가져와 병합
* git pull --rebase : 가져와서 병합하는 대신 내 커밋을 최신 상태 뒤로 재배치 (로그 깔끔)

9. 되돌리기 및 복구 (Undo, Reset, Restore)

git restore : 작업 중인 파일의 변경 사항 취소 (Checkout의 복구 기능 분리)
* git restore [파일] : 파일의 수정 내용을 모두 취소하고 마지막 커밋 상태로 되돌림
* git restore --staged [파일] : git add로 스테이징된 파일을 다시 내림 (Unstage)

git reset : 커밋을 취소하고 과거 시점으로 이동 (히스토리 삭제됨)
* git reset --soft HEAD~1 : 최신 커밋을 취소하되, 변경된 파일은 스테이징 상태로 보존 (커밋만 다시 하고 싶을 때)
* git reset --mixed HEAD~1 : (기본값) 커밋을 취소하고 스테이징도 내림 (작업 내용은 남음)
* git reset --hard HEAD~1 : (주의) 최신 커밋과 작업했던 내용까지 모두 영구 삭제

git revert : 기존 커밋을 삭제하지 않고, 반대되는 내용의 '새 커밋'을 생성 (협업 시 안전)
* git revert [커밋해시] : 해당 커밋의 내용을 취소하는 새로운 커밋 생성

10. 고급 도구 (Advanced Tools)

git stash : 작업 중인 코드를 임시로 다른 곳에 치워둠
* git stash : 현재 작업(스테이징+수정) 임시 저장
* git stash -u : 추적되지 않는(Untracked) 새 파일까지 포함해서 임시 저장
* git stash list : 저장된 스태시 목록 확인
* git stash pop : 가장 최근에 저장한 내용을 꺼내오고 목록에서 삭제
* git stash apply : 꺼내오기만 하고 목록에는 남겨둠 (계속 쓸 때)
* git stash drop : 가장 최근 스태시 삭제
* git stash clear : 모든 스태시 기록 삭제

git clean : 추적되지 않는 파일(Untracked)들을 강제로 삭제 (청소)
* git clean -f : 추적되지 않는 파일 강제 삭제
* git clean -fd : 추적되지 않는 파일과 디렉토리까지 모두 삭제 (새로 시작할 때 유용)

git tag : 특정 커밋에 버전 태그 달기 (배포 시 사용)
* git tag : 태그 목록 조회
* git tag [버전] : 현재 커밋에 Lightweight 태그 생성 (예: v1.0.0)
* git tag -a [버전] -m "메시지" : 설명이 포함된 Annotated 태그 생성
* git push origin [태그명] : 특정 태그 원격 업로드
* git push origin --tags : 로컬의 모든 태그 원격 업로드

Git Commit Type
* feat : 새로운 기능 추가
* fix : 버그 수정 - 잘못된 동작이나 오류 수정
* refactor : 코드 구조 개선 - 기능 변경 x, 함수분리, 변수 변경, 중복 코드 제거, 코드 구조 효율화
* style : 코드 포맷팅, 스타일 교정 - 코드 실행에 영향 없는 변경사항
* docs : 문서/주석만 변경 시, 코드 수정 X
* test : 테스트 코드 추가/수정/삭제
* chore : 기타 잡무, 빌드/설정 관리 - 코딩 외적인 작업
